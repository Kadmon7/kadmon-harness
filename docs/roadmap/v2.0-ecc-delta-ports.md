## Executive summary

- **Signal-to-noise is low by design**: ~85-90% of the catalog delta is out-of-Kadmon-scope
  padding (13 exotic language packs, community verticals: crypto, healthcare, homelab,
  supply-chain, marketing). Skills hit rate: 12 port-worthy of 216 new (5.5%). Agents/commands:
  ~12 signal files of 111.
- **Learning system: parity holds.** ECC continuous-learning v2.1 (project-scoped instincts,
  cross-project promotion) matches what Kadmon shipped 2026-04-19; Kadmon's decay is something
  ECC still lacks. Post-April delta is operational hardening — see the one exception below.
- **One proven fix for an open P1 bug**: ECC commit `f720885c` "archive observations only after
  successful analysis" is the same failure class as Kadmon's `/forge`-blind-on-long-sessions bug
  (BACKLOG P1, root cause `session-end-all.js` Phase 5 wipe). Their pattern: never clear the
  observations file on Stop; move it to an archive only after the analyzer consumed it.
- **AgentShield**: the scanner source is NOT in the ECC repo (separate repo, npm
  `ecc-agentshield`). Its 102 rules are regex/signature checks over harness config — a good
  config linter with excellent narrative, not a deep analysis engine (no AST taint, 21-entry
  CVE DB, admitted FP cluster). The **concept** — audit the `.claude/` tree as an attack
  surface — is the one layer Kadmon's stack (spektr, blocking hooks, permissions.deny) does not
  cover. Verdict: build the check natively (no external npm dependency); optionally run
  AgentShield once against our repos as third-party validation.
- **ECC moved reviewer agents to read-only tools** (July 2026, `f6e18e08`). Kadmon's reviewers
  carry Write/Edit. Worth a deliberate decision rather than inertia.

## Rejected with reason (so nobody re-litigates)

- 13 exotic language packs (65 rule files, 18 agents, ~20 commands) — Kadmon targets TS+Python.
- epic-*/orch-*/prp-* families — thin wrappers re-solving /abra-kdabra, /chekpoint, sprint.
- code-explorer/architect/simplifier, performance-optimizer, docs-lookup, /cost-report,
  /skill-health, /feature-dev, /project-init, /prune, /auto-update — weaker versions of
  existing Kadmon components.
- memory-persistence "hooks" — reference docs, not runtime; hooks/CATALOG.md already covers it.
- `common/code-review.md` — inferior to /chekpoint tiers.
- agentic-os / autonomous-agent-harness skills — describe what Kadmon already is.
- continuous-learning v2.1 wholesale — parity reached April 2026; only the small knobs above.
- react-performance skill — duplicates the installed vercel plugin (same upstream).
- Opus red/blue security pipeline — council skill already provides adversarial multi-agent.
- All multi-harness machinery (14 install targets, checked-in .codex/.cursor/... surfaces,
  ecc-universal packaging, Hermes/OpenClaw) — Kadmon is deliberately Claude-Code-only.
