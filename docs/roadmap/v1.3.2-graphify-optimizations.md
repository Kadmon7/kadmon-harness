## Scope (3 items)

### 1. Post-commit hook (auto-update grafo)

- [ ] **Accion**: `graphify hook install`
- [ ] **Verify**: `graphify hook status` reporta installed. Hacer dummy `.ts` commit + check `graphify-out/graph.json` mtime se actualizo
- [ ] **Pre-condicion verificada**: `.git/hooks/post-commit` no existe (Explore agent 2026-04-27) — zero conflict risk
- [ ] **Que hace**: post-commit detecta `.ts/.js` files cambiados via `git diff HEAD~1`, corre AST extraction sobre ellos, rebuilds `graph.json` + `GRAPH_REPORT.md`. Docs/imagenes ignora (necesita semantic, costoso) — usuario corre `/graphify --update` manual cuando aplique.
- **Severity**: LOW · **Effort**: S (~2 min) · **Cost**: $0

### 2. Git-log ADR linker (pivot: code→ADR backlinks via git history)

- [ ] **Problema original (descartado)**: grep ADR refs en `.ts/.js` files dio 0 matches. Code no cita ADRs inline.
- [ ] **Pivot**: parsear `git log --all --grep="ADR-\d+"` → para cada commit con ref ADR, extraer archivos tocados (`git show --name-only`) → crear edges `ADR-doc-node → code-file-node` etiquetadas `git_implemented_in` (relation nuevo, EXTRACTED 1.0).
- [ ] **Implementacion**: script `scripts/lib/graphify-git-adr-linker.py` (~50 lineas Python)
  - Lee `graphify-out/graph.json`, parsea git log, matchea ADR/plan nodes vs file nodes
  - Append edges, re-cluster, re-render via `graphify cluster-only` o equivalente
  - Backup `graph.json.pre-git-adr` antes de modificar
- [ ] **Verify**: contar nuevas `git_implemented_in` edges. Esperado: 30-80 (basado en `git log --grep="ADR" --oneline | wc -l`)
- [ ] **SKIP-condition**: si genera <10 edges, abortar (no vale el ruido)
- **Severity**: MEDIUM · **Effort**: S (~5 min) · **Cost**: $0 (deterministic)
