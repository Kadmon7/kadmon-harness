## Scope (7 items)

### Originales de v1.2.4 (3)

#### 1. `/medik --export` тАФ shareable redacted diagnostic file

- [ ] **Subcomando `--export`** en `.claude/commands/medik.md` que produce un archivo `diagnostic-YYYY-MM-DD-HHmm.txt` en el cwd
- [ ] **Contenido agregado**: ultimas N entries de `~/.kadmon/install-diagnostic.log` + ultimas N de `hook-errors.log` + output fresco de Check #9 + `platform`, `nodeVersion`, `git log -1`, branch
- [ ] **Redaction** antes de escribir: sustituir `os.homedir()` тЖТ `~`, rutas absolutas de proyecto тЖТ relativas, user-specific paths comunes (`C:\Users\<user>`, `/Users/<user>`, `/home/<user>`) тЖТ placeholder
- [ ] **No autenticacion, no red** тАФ solo I/O local. El usuario decide donde pegar el archivo (DM, GitHub issue, gist manual)
- [ ] **Tests**: redaction cubre los 3 casos (Windows, Mac, Linux) + verifica que el archivo es valido (parseable) + verifica que no contiene el home dir de la maquina de test

**Rationale**: middleground entre "copy-paste manual de stderr" (estado actual, friccion alta) y auto-submit a GitHub (descartado por privacy + spam + `gh auth` per collaborator). `/medik --export` = 80 % del valor con 10 % del riesgo. Si despues del shipping la friccion sigue, upgrade a `/medik report` con `gh gist create` + `gh issue create` consent-gated тАФ Sprint posterior.

#### 2. Schema `_v` field en `install-diagnostic.log` entries

- [ ] **`logInstallDiagnostic`** antepone `_v: 1` al serializar el `InstallHealthReport`. Output JSON: `{"_v":1,"rootDir":"...","platform":"...",...}`
- [ ] **`readInstallDiagnostics`** retorna entries con `_v` preservado para que consumers puedan feature-detect
- [ ] **Tests**: verifica que el field esta presente en todas las entries escritas post-v1.3 y que reader no borra entries legacy sin `_v` (defensive forward-compat)

**Rationale**: orakle MEDIUM (post-`/chekpoint` v1.2.3). Hoy reader retorna `Array<Record<string, unknown>>` y consumers deben defender campo-por-campo. Con `_v: 1` explicito, un futuro upgrade de schema puede migrar sin corromper historico.

#### 3. Typed reader wrapper тАФ `readTypedInstallDiagnostics()`

- [ ] **Nuevo export** en `scripts/lib/install-diagnostic-reader.ts` (TS, sibling de install-health.ts): `readTypedInstallDiagnostics(logDir?, limit?): Array<InstallHealthReport & { _v: number }>` тАФ best-effort cast con validacion de campos minimos (`rootDir`, `symlinks`, `timestamp`); entries sin esos campos se omiten con un warning a stderr
- [ ] **Consumer**: `/medik --export` lo importa para formatear el diagnostic. `/medik` Check #9 lo importa cuando mostremos "ultimos N reportes" en la salida
- [ ] **Tests**: fixtures con entries validas, corruptas, legacy (sin `_v`), mixto. Verifica que el array retornado tipa correctamente sin casts adicionales en el consumer

**Rationale**: orakle MEDIUM (post-`/chekpoint` v1.2.3). `readInstallDiagnostics` actual retorna `Record<string, unknown>` тАФ aceptable para el productor (session-start) pero penoso para consumers que necesitan acceder `report.symlinks[0].state`. Un wrapper tipado 1-shot evita que cada consumer reimplemente el type assertion.

Disparador explicito "post-Sprint-D cleanup antes de v1.2" ya cumplido.

#### 5. `/medik` Check #6 тАФ filtrar `.d.ts` files

**Origen** (v2.0 Hygiene backlog, diagnosticado por mekanik en /medik 2026-04-14 baseline): severidad LOW тАФ no afecta funcionalidad, solo es falso-WARN recurrente. Cabe en scope /medik expansion.
