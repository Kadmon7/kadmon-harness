## Sprint C: 2 BLOCKERS ‚Äî DONE 2026-04-14

> **Scope reducido 2026-04-13**: el sprint original tenia ~14 items entre DB hardening y observabilidad. Solo 2 bloqueaban plan-003 ‚Äî el resto se movio a v2.0 como "DB hardening + observabilidad secundaria". Ambos se cerraron el 2026-04-14 en commit `de9d4e0` via ADR-007 ([`ADR-007-sprint-c-data-integrity-fixes.md`](../decisions/ADR-007-sprint-c-data-integrity-fixes.md)). Plan-003 ya puede distribuir sin riesgo de acumular data corrupta.

- [x] **[BUG sistemico] `sessions.ended_at` < `started_at`** ‚Äî CERRADO 2026-04-14 (commit `de9d4e0`). Root cause: el resume branch de `startSession()` no limpiaba `session_end_state` ni `durationMs` antes del upsert, y `COALESCE` restauraba el `ended_at`/`duration_ms` previos produciendo la inversion. Fix: `startSession()` ahora llama `clearSessionEndState(id)` + borra `merged.durationMs` en ese branch. 19/35 sesiones corruptas (54%) reparadas via `scripts/migrate-fix-session-inversion.ts --apply`. Gotcha documentado en CLAUDE.md "Common Pitfalls".

- [x] **[BLOCKER] `hook_events.duration_ms` siempre NULL** ‚Äî CERRADO 2026-04-14 (commit `de9d4e0`). Root cause: `logHookEvent()` aceptaba el campo pero ningun hook lo calculaba. Fix: instrumentacion uniforme en los 9 blocking/warning hooks con `durationMs = Date.now() - start` al inicio de cada uno; JSDoc de `log-hook-event.js` ahora documenta `durationMs` como required. Verificacion: `SELECT COUNT(*) FROM hook_events WHERE duration_ms IS NOT NULL` > 0. Dashboard ya muestra Avg ms por hook en lugar de "‚Äî".
