## Sprint C: 2 BLOCKERS (solo lo que bloquea plan-003)

> **Scope reducido 2026-04-13**: el sprint original tenia ~14 items entre DB hardening y observabilidad. Solo 2 bloquean plan-003 — el resto se movio a v2.0 como "DB hardening + observabilidad secundaria". Estos 2 DEBEN arreglarse antes de distribuir el harness para evitar 2 semanas de data corrupta acumulada en 3 usuarios + migration retroactiva forzosa.

- [ ] **[BUG sistemico] `sessions.ended_at` < `started_at`** — diagnosticado 2026-04-13 via `getDbHealthReport()`. **19 de 35 sesiones (54%)** tienen timestamp inversion. Caso representativo: sesion `2e0d9b68` tiene `started_at=2026-04-12T23:54:40.953Z` y `ended_at=2026-04-12T23:50:33.492Z`. Rompe el calculo de duration en el dashboard (muestra valores absurdos como "30h 35m"). Root cause pendiente: `session-end-all.js` podria estar escribiendo `ended_at` con un timestamp de otra fuente (clean-exit.marker? session_start? zona horaria? reloj del sistema al resume?) en vez de `nowISO()` del momento del Stop hook. Reproducir: revisar `updateSession()` en `state-store.ts` y `persistSession()` en `session-end-all.js`. Considerar `CHECK (ended_at IS NULL OR ended_at >= started_at)` como guardrail de schema.

- [ ] **[BLOCKER] `hook_events.duration_ms` siempre NULL** — diagnosticado 2026-04-13 via `scripts/db-health-check.ts`. La columna existe en schema (`schema.sql:74`) pero `logHookEvent()` en `.claude/hooks/scripts/log-hook-event.js` no la esta poblando al escribir a `hook-events.jsonl`, o el extractor en `session-end-all.js` no la esta leyendo al persistir a DB. Efecto: dashboard muestra "—" en Avg ms para todos los hooks. Root cause pendiente: revisar si `logHookEvent()` recibe el campo desde los 9 hooks que lo llaman, o si se pierde en la extraccion JSONL→DB. Verificacion: `SELECT COUNT(*) FROM hook_events WHERE duration_ms IS NOT NULL` → 0/112.

**Plan de ataque proxima sesion**: `/abra-kdabra` con arkitect (analizar root cause de ambos en paralelo), konstruct (plan paso a paso), feniks (TDD — escribir tests que reproducen los bugs primero), implementacion minima, `/chekpoint full`. Tier full obligatorio: toca production code en `state-store.ts`, `session-end-all.js`, posiblemente `log-hook-event.js` y schema.
