~15 of 28 analyzed sessions did nothing but compact and exit. Use `claude --continue` or `claude --resume <id>` to keep context live without the ritual.

```

```

### Force a "fix, don't re-diagnose" loop
Friction data shows `repeated_failure_across_sessions` + `excessive_diagnosis_before_action`. Buggy code is your #1 friction (10 instances). Force action on known issues instead of re-exploring.

```
Before diagnosing anything, check memory/instincts for prior diagnoses of this same problem.
If found, skip re-diagnosis and attempt the smallest possible fix. Report what you changed
and whether it worked — don't re-explore.
```

### Lock in TDD + test-gated commits
Best sessions (152/152, 626/626, 180 tests passing) all involved TDD and test gates. Worst had buggy code and wrong approaches. Codify in `CLAUDE.md`:

```
For any non-trivial change: write or identify the failing test first, show it failing,
then implement, then show it passing. Do not commit unless the full suite is green —
report the test count in every commit message.
```
